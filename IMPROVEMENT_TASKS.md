# Agent Swarm 改进任务清单

## P0 - 关键问题（必须立即修复）

### P0-1: agent-daemon 缺少进程锁
**问题:** 可能启动多个 daemon 实例，导致冲突
**修复:** 添加 PID 文件检查和 flock

### P0-2: agent-watcher 没有错误恢复
**问题:** fswatch/inotify 崩溃后不会重启
**修复:** 添加 watchdog 和自动重启

### P0-3: 完成信号解析不够健壮
**问题:** 如果 agent 输出格式稍有变化就无法识别
**修复:** 使用正则表达式 + 多种格式支持

## P1 - 重要改进（应该尽快完成）

### P1-1: 添加并发控制
**问题:** 多个 agent 可能同时修改同一文件
**修复:** 使用 flock 文件锁

### P1-2: 改进日志格式
**问题:** 日志不够结构化，难以解析
**修复:** 使用 JSON 格式日志

### P1-3: 添加健康检查
**问题:** 无法知道 daemon/watcher 是否正常工作
**修复:** 添加 /health 端点或心跳文件

### P1-4: 超时机制
**问题:** agent 可能永久卡住
**修复:** 添加执行超时（默认 30 分钟）

## P2 - 优化改进（提升体验）

### P2-1: 添加通知系统
**功能:** 任务完成/失败时通知用户
**实现:** 支持 Slack/Discord/Email

### P2-2: Web Dashboard
**功能:** 可视化监控界面
**实现:** 简单的 HTML + SSE 实时更新

### P2-3: 指标收集
**功能:** 统计成功率、耗时、质量趋势
**实现:** 保存到 SQLite 数据库

### P2-4: Agent 模板系统
**功能:** 预定义的 agent 配置
**实现:** templates/ 目录 + 模板变量替换

## P3 - 未来增强（长期规划）

### P3-1: 分布式执行
**功能:** 多机器并行执行 agents
**实现:** 使用 Redis 作为任务队列

### P3-2: 成本优化
**功能:** 根据任务复杂度选择模型
**实现:** 简单任务用 GPT-3.5，复杂任务用 GPT-4

### P3-3: 智能调度
**功能:** 根据负载动态调度
**实现:** 优先级队列 + 资源监控

### P3-4: 多项目支持
**功能:** 同时管理多个项目
**实现:** 项目隔离 + 全局 orchestrator
